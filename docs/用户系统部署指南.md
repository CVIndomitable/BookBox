# BookBox V2 用户系统与网页版部署指南

## 已完成功能

### 1. 用户认证系统
- JWT token 认证（30 天有效期）
- 注册/登录/修改密码 API
- 用户信息管理（displayName, email, defaultSunDays, apnsToken）

### 2. 书库权限管理
- 三级角色：owner（所有者）/ admin（管理员）/ member（成员）
- 权限层级：owner > admin > member
- 功能：
  - 添加/移除成员
  - 修改成员角色（仅 owner）
  - 转让书库（owner → 新 owner，原 owner 降为 admin）
  - 退出书库（owner 不可退出，需先转让）
- 书库操作权限：
  - 查看：member 及以上
  - 编辑：admin 及以上
  - 删除：仅 owner

### 3. 晒书提醒系统
- 三级配置：用户全局（默认 365 天）→ 书库级别 → 箱子级别
- 提醒记录：targetType（library/box）+ targetId + nextSunAt
- APNs 推送集成（需配置证书）
- 定时任务：每小时检查一次到期提醒
- API：创建/更新/删除提醒，标记已晒书

### 4. 网页版前端
- React 18 + Vite + React Router
- 响应式设计（手机/电脑自适应）
- 已实现页面：
  - 登录/注册
  - 书库列表
  - 新建书库

## 部署步骤

### 后端部署

1. 安装依赖：
```bash
cd server
npm install
```

2. 配置环境变量（`.env`）：
```bash
# JWT 密钥（生产环境必须修改）
JWT_SECRET=your-secret-key-change-in-production

# APNs 配置（可选，用于推送）
APN_KEY_PATH=/path/to/AuthKey_XXXXXXXX.p8
APN_KEY_ID=XXXXXXXXXX
APN_TEAM_ID=XXXXXXXXXX
APN_TOPIC=com.yourapp.bookbox

# 晒书提醒检查间隔（毫秒，默认 1 小时）
SUN_REMINDER_INTERVAL_MS=3600000

# CORS 配置（网页版需要）
CORS_ORIGIN=http://localhost:5173,https://your-domain.com
```

3. 推送数据库变更：
```bash
npx prisma db push
```

4. 重启服务：
```bash
pm2 restart bookbox
```

### 网页版部署

1. 安装依赖：
```bash
cd web
npm install
```

2. 开发模式：
```bash
npm run dev
# 访问 http://localhost:5173
```

3. 生产构建：
```bash
npm run build
# 输出到 dist/ 目录，可用 nginx 托管
```

4. Nginx 配置示例：
```nginx
server {
    listen 80;
    server_name your-domain.com;

    root /path/to/web/dist;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api {
        proxy_pass http://localhost:3002;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

## API 文档更新

新增接口：

### 认证
- `POST /api/auth/register` - 注册
- `POST /api/auth/login` - 登录
- `GET /api/auth/me` - 获取当前用户
- `PATCH /api/auth/me` - 更新用户信息
- `POST /api/auth/change-password` - 修改密码

### 书库成员
- `GET /api/library-members/:libraryId/members` - 获取成员列表
- `POST /api/library-members/:libraryId/members` - 添加成员
- `PATCH /api/library-members/:libraryId/members/:userId` - 更新成员角色
- `DELETE /api/library-members/:libraryId/members/:userId` - 移除成员
- `POST /api/library-members/:libraryId/transfer` - 转让书库
- `POST /api/library-members/:libraryId/leave` - 退出书库

### 晒书提醒
- `GET /api/sun-reminders` - 获取我的提醒列表
- `POST /api/sun-reminders/library/:libraryId` - 为书库创建提醒
- `POST /api/sun-reminders/box/:boxId` - 为箱子创建提醒
- `PATCH /api/sun-reminders/:id` - 更新提醒
- `POST /api/sun-reminders/:id/mark-sunned` - 标记已晒书
- `DELETE /api/sun-reminders/:id` - 删除提醒

## 注意事项

1. **向后兼容**：旧的 API Token 认证方式保留，与 JWT 并存
2. **权限检查**：所有书库相关操作需先通过 `checkLibraryAccess` 中间件
3. **APNs 证书**：生产环境需配置真实的 APNs 证书和 Team ID
4. **JWT 密钥**：生产环境必须修改 `JWT_SECRET`，建议 32 字节随机字符串
5. **CORS 配置**：生产环境需在 `CORS_ORIGIN` 中添加网页版域名

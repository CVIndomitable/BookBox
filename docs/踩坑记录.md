# 踩坑记录

## 新增带大请求体的路由时忘记设置 express.json limit

**现象**：接口返回 413 Payload Too Large。

**根因**：`server/src/index.js` 中按路径前缀设置了不同的 JSON 请求体大小限制：

```js
app.use('/api/llm/recognize', express.json({ limit: '10mb' }));
app.use('/api/books', express.json({ limit: '2mb' }));
app.use(express.json({ limit: '1mb' })); // 兜底
```

`express.json()` 是按路径前缀匹配的。新增路由（如 `/api/llm/extract-book-details`）如果路径不以已有的高限制前缀开头，就会落到兜底的 1MB 限制上，携带 base64 图片时必然 413。

**教训**：新增任何接收 base64 图片或大 JSON 的端点时，同步检查是否需要在 `index.js` 中为其添加独立的 `express.json({ limit: 'Xmb' })` 中间件。

**涉及路由**：
- `POST /api/llm/extract-book-details` — 拍照补充详情（2026-05-02 修复）

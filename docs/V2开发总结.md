# BookBox V2 开发总结

## 已完成功能

### 1. 用户认证系统 ✅
- JWT token 认证（30 天有效期）
- 注册/登录/修改密码 API
- 用户信息管理（displayName, email, defaultSunDays, apnsToken）
- 密码加密存储（bcrypt）

### 2. 书库权限管理 ✅
- 三级角色：owner（所有者）/ admin（管理员）/ member（成员）
- 权限层级控制：owner > admin > member
- 成员管理：添加/移除/修改角色
- 书库转让功能
- 退出书库功能
- 所有书库操作都有权限检查

### 3. 晒书提醒系统 ✅
- 三级配置：用户全局（默认 365 天）→ 书库级别 → 箱子级别
- 提醒记录管理：创建/更新/删除/标记已晒书
- APNs 推送集成（需配置证书）
- 定时任务：每小时检查一次到期提醒
- 推送通知包含跳转信息

### 4. 网页版前端 ✅
- **技术栈**：React 18 + Vite + React Router
- **响应式设计**：适配手机和电脑
- **已实现页面**：
  - 登录/注册页面
  - 书库列表（首页）
  - 书库详情页（书籍/成员/导入导出/统计）
  - 晒书提醒管理页面
- **核心功能**：
  - 用户认证与 token 管理
  - 书库 CRUD 操作
  - 书籍搜索与筛选
  - 成员管理（添加/移除）
  - CSV 批量导入/导出书籍
  - 晒书提醒管理

### 5. 后端 API 增强 ✅
- 用户认证路由（`/api/auth/*`）
- 书库成员管理路由（`/api/library-members/*`）
- 晒书提醒路由（`/api/sun-reminders/*`）
- 书籍批量导入/导出接口
- 权限中间件（authenticate, checkLibraryAccess）
- APNs 推送服务

## 数据库变更

### 新增表
- `users` - 用户表
- `library_members` - 书库成员关联表（多对多）
- `sun_reminders` - 晒书提醒表

### 修改表
- `libraries` - 新增 `sun_days` 字段（书库级别晒书间隔）
- `boxes` - 新增 `sun_days` 字段（箱子级别晒书间隔）

### 新增枚举
- `MemberRole` - owner/admin/member

## 技术亮点

1. **权限系统设计**：
   - 中间件链式调用：authenticate → checkLibraryAccess(role)
   - 角色层级控制，灵活的权限检查
   - 所有书库操作都经过权限验证

2. **晒书提醒设计**：
   - 三级配置继承：用户 → 书库 → 箱子
   - 定时任务 + APNs 推送
   - 提醒状态管理（notified 标记）

3. **前端架构**：
   - Context API 管理全局认证状态
   - Axios 拦截器统一处理 token 和错误
   - 响应式 CSS，移动端优先

4. **安全性**：
   - JWT token 认证
   - bcrypt 密码加密
   - CORS 白名单配置
   - 401 自动退出登录

## 部署注意事项

1. **环境变量配置**（`.env`）：
   ```bash
   JWT_SECRET=your-secret-key-change-in-production
   APN_KEY_PATH=/path/to/AuthKey_XXXXXXXX.p8
   APN_KEY_ID=XXXXXXXXXX
   APN_TEAM_ID=XXXXXXXXXX
   APN_TOPIC=com.yourapp.bookbox
   SUN_REMINDER_INTERVAL_MS=3600000
   CORS_ORIGIN=http://localhost:5173,https://your-domain.com
   ```

2. **数据库迁移**：
   ```bash
   cd server
   npm install
   npx prisma generate
   npx prisma db push
   ```

3. **网页版构建**：
   ```bash
   cd web
   npm install
   npm run build
   # 输出到 dist/ 目录
   ```

4. **Nginx 配置**：
   - 静态文件托管（web/dist）
   - API 反向代理（/api → localhost:3002）

5. **APNs 证书**：
   - 生产环境需配置真实证书
   - 在 Apple Developer 后台生成 .p8 文件
